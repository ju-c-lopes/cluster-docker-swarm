Automação de Cluster Docker Swarm com Vagrant

Desafio feito para o curso de Docker Fundamentals onde foi definido um cluster de máquinas virtuais com o Docker pré instalado, definindo o volume para a aplicação, o nó manager na máquina 01, nomeada de Master, e os nós workers nas demais máquinas.

Para o desafio, foi utilizado o Vagrant, que define infraestrutura por código, configurado em um vagrant file, com todas as configurações do cluster já definidas e com a aplicação pronta para ser acessada.

A aplicação é um servidor apache que disponibiliza uma aplicação web na porta 80:80 e pode ser acessada através dos IPs das máquinas virtuais que terão os containers replicados.
